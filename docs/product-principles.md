# Product Principles

## Product Definition

Waiting For AI is a simple app for capturing things a user is waiting on and helping them decide when to check again.

## User Pain Solved

People lose track of externally pending items because the information arrives in messy formats and the next follow-up time is unclear. The product reduces this by turning scattered inputs into a structured list of waiting items.

## Why Waiting Is Different From Todo

A todo item is usually an action the user owes. A waiting item is blocked on someone or something else. The user may need to follow up later, but the primary state is external dependency, not self-assigned work.

## App Behavior Principles

- Conservative reminders: only suggest follow-up timing that feels calm and credible.
- Evidence-based extraction: derive fields from actual input rather than inventing confident summaries.
- Ambiguity goes to review: uncertain extraction should surface for user confirmation.
- Minimal input burden: the app should ask for the least effort necessary to create a useful waiting item.
